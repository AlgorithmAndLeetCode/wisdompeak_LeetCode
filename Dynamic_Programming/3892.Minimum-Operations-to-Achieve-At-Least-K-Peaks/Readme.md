### 3892.Minimum-Operations-to-Achieve-At-Least-K-Peaks

很明显，当长度n为偶数时，我们最多可以搞出n/2个peak；当长度n为奇数时，我们最多也只能搞出n/2个peak（因为不可能有两个相邻的peak）。

在不考虑环形的约束时，本题本质就是在长度为n的数组里挑k个元素，并且挑选的元素不能相邻，求最小的总代价。考虑到n和k都不超过5000，这明显就是一个o(n*k)数量级的动态规划。此题和house robber非常相似：令dp[i][j]表示在前i个元素里挑j个，决策关键就是第i个元素是否选择。转移方程是:
```
dp[i][j] = min(dp[i-1][j], dp[i-2][j-1]+cost[i]).
```

现在考虑环形的约束。其实这和213一样有常规的套路。就是考虑第一个元素是否选。如果第一个选的话，那么就是再从第三个到倒数第二个元素里面选择k-1个。如果第一个不选的话，那么就是从第二个到最后一个元素里面选择k个。两个分支都可以用同一个DP函数解决。
